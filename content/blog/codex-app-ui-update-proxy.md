最近遇到一个 Codex App UI 无法更新的问题，核心原因是代理范围不完整。

## 现象

- Codex App 启动后 UI 没有更新。
- 仅代理了某个进程（或局部链路）时，更新一直失败。

## 原因

Codex App 启动时，可能会拉起 Electron 相关框架或其他子进程来完成 UI 更新。  
如果只给主进程做代理，子进程网络请求可能仍然直连，导致更新失败。

## 解决

- 开启系统代理后，UI 更新成功。
- 经验上应避免只代理单一进程。

## 建议

1. 尽量使用全局代理或系统级代理。
2. 至少确保主进程和子进程都能走同一代理出口。
3. 排查时优先检查 Electron 子进程的网络连通性。

这样可以明显降低 UI 更新失败的概率。
